This website is running in a kubernetes cluster with 3 Raspberry Pi, seriously!

Yeah, I know this is like using a bazooka to kill an ant, but I made this mainly to learn kubernetes and now I have a cluster and some basic infrastructure that make things easier to put some personal apps available through the internet without having to pay new servers.

Before starting this project I knew I was going to learn some new things but the learning curve went way beyond what I was imagining it would be. I learned a lot about networks, kubernetes, docker, jenkins, linux and even how to get out of vi.

Check out my boys (rocket-raccoon, groot and star-lord). They are 3 Raspberry Pi 4B with 8GB ram each, running Raspberry Pi OS (64 bit).

Everytime I push to the GitHub repository of this project, a webhook is sent to my Jenkins (which is running on rocket-raccoon) and automaticaly starts a build and push the result image to my dockerhub repository



In the next image we can see an rolling update, when new pods are created before terminating the older ones, this way I can update the application with 0 downtime.

This was a super fun project to do and it was totally worth it.
